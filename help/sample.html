Some information:

  1. Internally all samples are kept in the format:

    • 44100 khz

    • 2 channel

    • 16 bit

    • signed

    • little endian

    this is what you see in the sample display window (left channel)

  2. Recording is done with the settings you enter to the right
    (and is than converted to above format)

  3. (the original) samples can be saved, save format is "wav", saving is done in the above format, since in this format all information is kept (internally) anyway.

  4. (the vectrex) samples can be saved, save format is "wav", saving is done in the format chosen (only sample rate can be chosen, the rest is allways set).

  5. you can save the vectrex samples as "raw" data, the raw data uses 8 bit signed data (this is what vectrex wants)

  6. Normalization uses no range compression algorithm, this is just a maximization so that the largest absolut sample reaches its format maximum (and all other samples are scaled accordingly)
